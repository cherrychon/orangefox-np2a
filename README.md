## Source layout

- \`device_nothing_pacman/\` - device tree for pacman with OrangeFox/TWRP 12.1 support
- \`installer/\` - simple vendor_boot-based installer ZIP (flashes vendor_boot to the active slot)

## Base sources

- Recovery tree: OrangeFox R14.1 (ported on top of TWRP 12.1)
- Manifest (TWRP 12.1):
  https://github.com/minimal-manifest-twrp/platform_manifest_twrp_aosp.git (branch: twrp-12.1)

## Quick build instructions

```bash
# 1) Create working dir
mkdir -p ~/ofx-12 && cd ~/ofx-12

# 2) Init & sync TWRP 12.1 manifest
repo init -u https://github.com/minimal-manifest-twrp/platform_manifest_twrp_aosp.git -b twrp-12.1
repo sync -c --no-tags --no-clone-bundle -j4

# 3) Clone OrangeFox recovery
git clone https://gitlab.com/OrangeFox/bootable/Recovery.git -b fox_14.1 bootable/recovery

# 4) Clone this repo (device + installer)
git clone https://github.com/YOUR_GITHUB_USERNAME/orangefox-pacman.git ofx-local

# 5) Copy device tree
cp -a ofx-local/device_nothing_pacman device/nothing/pacman

# 6) Build
source build/envsetup.sh
lunch twrp_pacman-eng
mka -j2 vendorbootimage

#The built image will be in:

`out/target/product/pacman/vendor_boot.img`

#Flash (at your own risk) or Boot(recommended method of use right now):

fastboot flash vendor_boot vendor_boot.img
# or
fastboot boot vendor_boot.img

#Installer ZIP
#The `installer/` directory contains a simple shell-based installer that:
#extracts `vendor_boot.img` from the ZIP
#detects the active slot via `ro.boot.slot_suffix`
#flashes `/dev/block/bootdevice/by-name/vendor_boot_<slot>`

#You can rebuild the installer ZIP with:

cd installer
zip -r9 ../OrangeFox-pacman-vendor_boot-installer.zip .
